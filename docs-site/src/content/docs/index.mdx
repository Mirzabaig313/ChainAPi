## Stop copying tokens between Postman tabs

Modern SaaS backends are multi-actor. To test "admin approves a customer
refund", you need to log in as three actors and create six prerequisite
records first. ChainAPI treats your API as a dependency graph: define
each actor and resource **once**, then run any endpoint with a single
command — the engine auto-resolves the entire chain.

```bash
chainapi run refund.approve --project marketplace
# resolves: vendor.login → customer.verify_otp →
#           product.create → product.publish →
#           order.create → order.pay →
#           refund.request → refund.approve
# 9 steps. 3.4 seconds. Zero copy-paste.
```

## Quick install

<Tabs>
    <TabItem label="macOS (Homebrew)">
        ```bash
        # Coming with v1 release. Today: build from source — see
        # /dev/building/
        brew install chainapi/tap/chainapi
        ```
    </TabItem>
    <TabItem label="Linux">
        ```bash
        # AppImage:
        curl -L https://github.com/chainapi/chainapi/releases/latest/download/chainapi-linux-x86_64.AppImage -o chainapi
        chmod +x chainapi
        ```
    </TabItem>
    <TabItem label="Windows">
        ```powershell
        # Scoop:
        scoop bucket add chainapi https://github.com/chainapi/scoop-bucket
        scoop install chainapi
        ```
    </TabItem>
    <TabItem label="Build from source">
        ```bash
        git clone https://github.com/chainapi/chainapi
        cd chainapi
        cmake --preset macos-debug
        cmake --build --preset macos-debug
        ```
        See [Building from source](/dev/building/) for the full prerequisites.
    </TabItem>
</Tabs>
